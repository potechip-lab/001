# 大学ニュースモニタリングサイト

大学名・部活・OB選手名などのキーワードに関するニュースを毎日自動収集し、  
ステークホルダー向けに公開するダッシュボードです。  
**すべて無料サービスで動作します（サーバー不要）。**

---

## ファイル構成

```
リポジトリルート/
├── index.html          ← 公開されるウェブページ
├── keywords.json       ← 監視するキーワードの設定ファイル
├── fetch_news.py       ← ニュース取得スクリプト
├── data.json           ← 取得した記事データ（自動生成）
└── .github/
    └── workflows/
        └── update.yml  ← GitHub Actions 自動実行設定
```

---

## セットアップ手順

### 1. GitHubリポジトリを作成する

- GitHub で新しい **Public** リポジトリを作成します
- 上記のファイルをすべてリポジトリに追加します
- `update.yml` は `.github/workflows/` フォルダの中に置いてください

### 2. キーワードをカスタマイズする

`keywords.json` を編集して、対象の大学名・部活・選手名に書き換えます。

```json
{
  "university_name": "○○大学",        ← 表示用の大学名
  "match_keywords": [                  ← トレンドと照合するキーワード
    "○○大学",
    "田中太郎"
  ],
  "categories": {
    "大学全般": [                       ← カテゴリ名は自由に変更可
      "○○大学",
      "○○大学 入試"
    ],
    "部活・スポーツ": [
      "○○大学 サッカー部",
      "○○大学 野球部"
    ],
    "OB・プロ選手": [
      "田中太郎",
      "佐藤花子"
    ]
  }
}
```

**tips:**
- 選手名は「田中太郎　サッカー」のように競技名を加えると精度が上がります
- カテゴリは何個でも追加できます

### 3. GitHub Pages を有効にする

1. リポジトリの **Settings** タブを開く
2. 左メニューの **Pages** をクリック
3. Source を **"Deploy from a branch"** に設定
4. Branch を **main**、フォルダを **/ (root)** に設定して **Save**

これで `https://ユーザー名.github.io/リポジトリ名/` でサイトが公開されます。

### 4. 初回データを取得する

最初は `data.json` がないので、GitHub Actions を手動で実行します。

1. リポジトリの **Actions** タブを開く
2. 左メニューの **Update News Data** をクリック
3. **Run workflow** ボタンを押す
4. 数分後に `data.json` がコミットされます

以降は **毎日 06:00 JST** に自動実行されます。

## カスタマイズ

### 更新時刻を変える

`update.yml` の cron 行を変更します（UTC表記）。

```yaml
- cron: "0 21 * * *"   # 06:00 JST（UTC+9）
- cron: "0 0 * * *"    # 09:00 JST に変えたい場合
```

### 取得件数を変える

`fetch_news.py` の `max_items` を変更します（デフォルト: 8件）。

```python
articles = fetch_google_news_rss(keyword, max_items=8)
```

---

## 注意事項

- Google News RSS は非公式の仕様です。Googleの仕様変更により取得できなくなる場合があります
- キーワード数が多い場合（20件以上）、取得に時間がかかります
- 記事リンクは各メディアのサイトに遷移します（本サービスは記事を転載しません）

| GitHub Pages | 月100GBまで |
| Google News RSS | 認証不要・無料 |
| Google Trends RSS | 認証不要・無料 |
